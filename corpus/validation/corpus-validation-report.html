Fehlerklassen K1–K8

Epistemic failure patterns. The harness validates per class, never per trap ID — which is exactly how the 3 blind traps get checked without being known.

K1 Grün-aber-falsch (Green but wrong)

7

Probe passes, claim is mechanically consistent, but semantically false. Sign convention misread, FX pair inverted, IC revenue not excluded, rebate accrual forgotten— the books can still close perfectly while answers are wrong.

Tool consequenceForces the Silent-Wrong-Answers = 0 metric. For Z1–Z4, reference results are known; any deviation without a flagged, documented assumption is a release blocker. Requires V3 to extract domain rules from documents (F14/F15/F19 can only be resolved from buchhaltungsrichtlinie.pdf), not silent inference.

F5F6F14F19F22F27BLIND_1

K2 Doppelzählung durch Strukturbrüche (Double-counting via structural breaks)

4

Partial delivery (1 order → N invoices) and blanket invoicing (N orders → 1 invoice) create fan-out pitfalls. Reversals as pair-linked records. Legacy credit notes living in two places (D7 until 2024-06, then type 'G' in invoices).

Tool consequenceRequires join logic that respects the cardinality and reversal semantics. Naive SUM over invoices double-counts. Requires concept claims for "credit memo" and "reversal pair" to make join logic explicit and testable.

F2F3F4F20

K3 Konventionssemantik (Convention semantics)

8

Domain rule lives as a pure convention: Haben is negative, revenue accounts are 4000–4999 minus 4800-range erlösschmälerungen, FX conversion strategy selects M-rates not B-rates. No data field encodes these; they live in the accounting policy document only.

Tool consequenceInferences alone cannot discover these—they require documents (V3) and become concept claims backed by policy references. Reconciliation against known-true numbers (Z2/Z3) then proves the chosen rule was right.

K4 Nicht-wertbasierte Strukturen (Non-value-based relationship structures)

9

Join keys that are not value overlap: validity predicates (customer_hierarchy has valid_from/valid_to), PLZ ranges (territory via BETWEEN), positional strings decoded by function, not lookup (product hierarchy).

Tool consequenceCandidate matrix is blind to these (no value overlap to seed hypotheses). Requires three new probe templates: validity_join, range_join, decode. Rule: new templates only if a corpus trap mandates them; otherwise defer as a fachfrage.

K5 Invariantensonden (Invariant probes)

0

Correctness constraints that are landscape-bound, not claim-bound. Balance sheet closes (debits = credits per entity and period), subledger = GL totals, IC transactions symmetric across entities, no cash unapplied without documented reason.

Tool consequenceA second class of probes that exist without an associated claim. They live in the domain's own invariant catalog and are run as part of the epistemics engine's consistency checks, not as evidence for a user-authored claim. New system metric: all invariants must hold or explicitly emit a gap/finding.

Landscape-bound — no claim-tagged traps. Validated by the invariant harness: balance closes, subledger = GL, IC symmetry.

K6 Legitime Waisen (Legitimate orphans)

3

Orphans (records without a parent in the join) that are not errors. Open orders with no invoice (legitimate wait state), AR activities without a customer in master (prospecting), returns dated before the order book (data lag).

Tool consequenceVerdict granularity: a claim "orders have invoices" fails the probe but does not get marked contradicted—instead generates a fachfrage "what about open orders?" Orphan ≠ error; the claim stays inferred and a question surfaces. The system must not auto-promote over legitimate doubt.

F1F10F13

K7 Vergiftete Anker und Restatements (Poisoned anchors and restatements)

4

A document number that must never be found or reconciled (divested business revenue in an old press release), a restated prior-year number that contradicts all current data (deliberate intentional mismatch for testing), a number appearing only in a chart graphic (no text anchor, structurally weak evidence).

Tool consequenceReconciliation needs negative discipline: not every document number gets to be found. V3 must flag chart-only numbers as low-confidence. Restatements must be explicitly marked as such; no silent replacement of old data with new. Claims based on poisoned anchors must never be promoted.

F23F24F26BLIND_3

K8 Tell-Statements (unverifizierbar)

2

Scripted domain statements played during test runs (tell_statements.yaml). Not in the K1–K7 spec catalog — introduced by the generator; unverifiable against data, must remain business-confirmed-only claims.

Tool consequenceA tell without data support may never be silently promoted to tested; it stays a business-confirmed claim with its utterance as the only evidence.

Not part of the spec catalog (trap_classes.yaml) — generator-introduced class.
